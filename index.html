🎯 Problema & Necessidades

📌 Problema identificado

Proprietários e pequenas imobiliárias enfrentam dificuldades para gerenciar aluguéis de forma organizada: cadastro de inquilinos desatualizado, emissão manual de boletos e controle de pagamentos ineficiente, gerando inadimplência e retrabalho.

✔ Solução: Aplicativo mobile "Aluga Gestão" que centraliza cadastro de inquilinos, propriedades, emite boletos mensais e permite baixa rápida de pagamentos.

✅ Requisitos Funcionais

  • RF01 - CRUD completo de inquilinos (nome, CPF, contato, endereço).
  • RF02 - CRUD de imóveis (endereço, valor do aluguel, data de vencimento).
  • RF03 - Gerar boleto mensal automaticamente para cada contrato ativo.
  • RF04 - Registrar baixa de pagamento (data, valor pago, forma de pagamento).
  • RF05 - Listar inadimplentes e enviar alertas automáticos.
  • RF06 - Relatório de receitas por período (mês/ano).

⚙️ Requisitos Não-Funcionais

  • RNF01 - Aplicativo mobile (Android/iOS) com interface simples e rápida.
  • RNF02 - Dados criptografados em nuvem (Firebase/Supabase) com sincronização.
  • RNF03 - Geração de boletos em PDF com código de barras (API externa).
  • RNF04 - Tempo de resposta < 1.5s para operações críticas.

🔄 Modelo de Procedimentos
1
Cadastro de Inquilino e Imóvel
Usuário acessa "Cadastros" → preenche dados do inquilino (nome, CPF, contato) e endereço do imóvel, valor do aluguel, dia de vencimento → sistema salva no banco → vincula inquilino ao imóvel automaticamente.
2
Emissão Mensal de Boletos
Todo dia 1º de cada mês (ou manualmente), sistema verifica contratos ativos → gera boleto com valor do aluguel + data_vencimento (dia definido) → armazena código de barras e status "pendente". Usuário pode emitir PDF e compartilhar via WhatsApp.
3
Baixa de Pagamento
Inquilino paga → usuário acessa tela "Pagamentos", localiza boleto pendente, clica em "Dar Baixa" → insere data, valor, forma de pagamento → sistema atualiza status do boleto para "pago" e registra na tabela Pagamento.
4
Controle de Inadimplência e Relatórios
Sistema compara data atual com vencimentos não pagos → exibe lista de inadimplentes → opção de enviar notificação automática. Gera relatório de receitas por período (mês/ano) agrupado por imóvel/inquilino.
🔁 Procedimento (pseudocódigo - Baixa de Pagamento):
function registrarBaixa(id_boleto, valorPago, formaPagamento):
  if boleto.status == 'pendente' and valorPago == boleto.valor:
    INSERT INTO Pagamento VALUES (...);

    return "Baixa registrada com sucesso";
  else:
    return "Valor não confere ou boleto já pago";
